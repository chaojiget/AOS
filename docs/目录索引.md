# 项目目录索引

本索引帮助新成员快速了解 AOS 仓库的最终目录规划与各子目录职责。配合《需求全景规划》与《详细设计与 API 契约》阅读，可快速定位到对应实现与文档。

## 顶层目录速览

| 目录 | 说明 | 状态 |
| ---- | ---- | ---- |
| `AGENTS.md` | 面向 AI/人类协作者的操作手册，定义默认工作流、质量门禁与守护策略。 | ✅ 已提供 |
| `README.md` | 仓库概览入口，指向文档索引与快速导航。 | ✅ 已提供 |
| `docs/` | 所有需求、设计、流程与操作文档。 | ✅ 已提供 |
| `core/` | Agent 内核：循环控制、上下文建模、事件契约与核心类型。 | 🛠️ 持续迭代 |
| `runtime/` | Episode 写入、回放引擎、守护与指标落地脚本。 | 🛠️ 持续迭代 |
| `adapters/` | LLM/MCP/外部服务适配层，统一工具与模型接口。 | 🛠️ 持续迭代 |
| `packages/` | 可复用包（技能流水线、提示词、schema、评测脚本等）。 | 🛠️ 持续迭代 |
| `servers/` | NestJS/Next API 服务、SSE/WS 推送与回放接口实现。 | 🛠️ 持续迭代 |
| `ui/` | 前端组件与页面（聊天、日志、技能、Guardian 面板）。 | 🛠️ 持续迭代 |
| `components/` | 跨页面复用的 React 组件与样式。 | 🛠️ 持续迭代 |
| `pages/` | Next.js 页面路由（SSR/SSG 入口）。 | 🛠️ 持续迭代 |
| `lib/` | 通用工具函数（格式化、校验、请求封装）。 | 🛠️ 持续迭代 |
| `config/` | 配置文件、环境变量模板、MCP 注册表。 | ✅ 已提供 |
| `scripts/` | CLI/评估脚本（如 `skills-eval`、数据迁移、回放检查）。 | ✅ 已提供 |
| `tests/` | 单元、集成与回放测试集合。 | ✅ 已提供 |
| `locales/` | i18n 资源。 | ✅ 已提供 |
| `runtime/episodes` | 运行日志 JSONL 存储目录（如启用）。 | ⏳ 运行时生成 |
| `reflect/` | 代理回放与调试素材。 | ✅ 已提供 |

> ✅ 表示目录已存在且可直接使用；🛠️ 表示正在建设或持续演进；⏳ 表示运行或构建时生成的动态目录。

## `docs/` 子目录与作用

| 子路径 | 说明 |
| ------ | ---- |
| `docs/需求全景规划.md` | 最终需求基线，总览阶段目标、功能拆解、验收与风险。 |
| `docs/SRS.yaml` | 结构化需求说明，用于自动校验与追踪需求变更。 |
| `docs/decision.md` | Discovery 阶段的方案比较、决策结论与风险缓解。 |
| `docs/详细设计与API契约.md` | 架构分解、模块职责、关键数据结构与 API 契约。 |
| `docs/PROJECT.md` | 项目治理、Stage-Gate、RACI 与协作规范。 |
| `docs/操作指南_Episodes与Replay.md` | Episodes 生产、回放与审计操作指南。 |
| `docs/需求草稿/` | v0/v1 草稿及历史背景文档，保留决策脉络。 |

## 新成员入门路径

1. 阅读 `AGENTS.md` 与 `docs/需求全景规划.md`，掌握项目目标、范围与守护要求。
2. 浏览本索引，熟悉代码与文档目录定位。
3. 深入 `docs/详细设计与API契约.md` 与 `docs/PROJECT.md`，了解实现细节与治理流程。
4. 按需查阅 `docs/操作指南_Episodes与Replay.md`，亲自跑通 Episode → Replay 闭环。
5. 若需追溯背景或设计演进，可阅读 `docs/需求草稿/` 下的历史版本。

> 建议在新增目录或模块时，同步更新本索引，以保持“文档即基线”的一致性。
